boolean breakBlock ( Player  player,
int  x,
int  y,
int  z 
)

Simulate a player breaking a block in the world.

If the player is in org.bukkit.GameMode#CREATIVE game mode, the block is broken immediately. Otherwise the org.bukkit.event.player.PlayerInteractEvent is called first followed by a 10 tick delay before calling the org.bukkit.event.block.BlockBreakEvent and breaking the block

The 10 tick delay is a thread pause so there is no need to schedule a task to handle the result. When the method ends the entire event cycle is finished. The Bukkit scheduler service is still running during the 10 tick pause.

Parameters
playerThe player.
xThe X coordinates of the block.
yThe Y coordinates of the block.
zThe Z coordinates of the block.
Returns
True if not cancelled by events.
